# Polityka bezpieczeństwa

## Zgłaszanie podatności

Nie publikuj szczegółów podatności w publicznym issue. Użyj prywatnego zgłoszenia w sekcji Security Advisories repozytorium.

Nie dołączaj sekretów, danych dostępowych, prywatnych listingów, zdjęć klientów, podpisanych URL-i ani innych danych osobowych. Do reprodukcji używaj minimalnych danych syntetycznych.

## Zakres wsparcia

Poprawki bezpieczeństwa dotyczą aktualnej wersji gałęzi `main`. Starsze rewizje nie otrzymują osobnych poprawek.

## Odpowiedzialne ujawnienie

Po otrzymaniu kompletnego zgłoszenia maintainer potwierdza problem, ocenia wpływ, przygotowuje poprawkę i publikuje informację dopiero po udostępnieniu bezpiecznej wersji.
